The Third-Party Service Provider Management domain ensures that third-party service providers (TPSPs) handling cardholder data (CHD) or impacting the security of the Cardholder Data Environment (CDE) are appropriately managed and compliant with PCI DSS requirements. The evidence below is consolidated for functional stakeholders (e.g., compliance, procurement, IT teams), using practical, real-world artifacts aligned with PCI DSS v4.0.1 requirements.

Evidence Requirement PCI DSS Requirement(s) Description and Practical Examples
TPSP Inventory 12.8.1 A maintained list of all TPSPs involved in storing, processing, or transmitting CHD, or impacting CDE security, including descriptions of their services. Example: An Excel spreadsheet listing TPSPs (e.g., "Payment Gateway X, processes transactions; Cloud Provider Y, hosts CDE servers"), with details on service scope and contact information.
TPSP Written Agreements 12.8.2 Written agreements with TPSPs acknowledging their responsibility for securing CHD and providing information for PCI DSS compliance. Example: A PDF contract or service-level agreement (SLA) with a payment processor, including clauses on PCI DSS compliance, data security obligations, and audit rights.
TPSP Due Diligence Records 12.8.3 Documentation of due diligence performed before engaging TPSPs, evaluating their PCI DSS compliance, security practices, and incident response capabilities. Example: A PDF due diligence report for a cloud provider, summarizing their PCI DSS AOC review, security certifications (e.g., ISO 27001), and breach notification processes.
TPSP Compliance Status Monitoring 12.8.4, 12.8.5 Records of annual monitoring of TPSPs' PCI DSS compliance status, including Attestations of Compliance (AOCs) or other evidence of compliance. Example: A folder containing AOCs from TPSPs (e.g., payment gateway's AOC dated within 12 months) and an Excel tracker noting compliance status and review dates.
Shared Responsibility Matrix 12.9.1, 12.9.2 Documentation detailing which PCI DSS requirements are managed by each TPSP, which are managed by the entity, and any shared responsibilities. Example: An Excel matrix listing PCI DSS requirements (e.g., Requirement 3 for data protection), with columns indicating "TPSP" (e.g., cloud provider encrypts data), "Entity" (e.g., manages key custodians), or "Shared" (e.g., joint monitoring).
TPSP Compliance Support Evidence 12.9.2 Evidence that TPSPs provide information to support the entity's PCI DSS compliance, such as compliance reports or audit data. Example: A PDF report from a TPSP (e.g., cloud provider's SOC 2 report) shared with the entity, detailing security controls relevant to CDE hosting, plus email correspondence confirming data access for audits.
