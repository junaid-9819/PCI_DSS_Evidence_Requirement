The Physical Security domain ensures that physical access to cardholder data (CHD) and the Cardholder Data Environment (CDE) is restricted and monitored to prevent unauthorized access. The evidence below is consolidated for functional stakeholders (e.g., facilities, security, IT teams), using practical, real-world artifacts aligned with PCI DSS v4.0.1 requirements.

Evidence Requirement PCI DSS Requirement(s) Description and Practical Examples
Physical Access Control Policy 9.1.1, 9.1.2 A documented policy outlining processes for restricting physical access to CDE areas (e.g., data centers, server rooms) and sensitive areas, including authorization and monitoring procedures. Example: A PDF policy document titled "Physical Security Policy" detailing access controls for the CDE data center, including badge access requirements and visitor management procedures.
Physical Access Control Configurations 9.1.3, 9.1.4, 9.2.1, 9.2.2 Configurations or logs showing implemented controls (e.g., badge readers, biometric locks) to limit and monitor access to CDE areas, ensuring only authorized personnel are granted entry. Example: Screenshots from an access control system (e.g., Lenel, HID) showing badge reader settings for a CDE server room, plus access logs listing authorized personnel entries.
Visitor Management Records 9.3.1, 9.3.2, 9.3.3 Records of visitor access to CDE areas, including identification, authorization, and escort procedures, with logs maintained for at least 3 months. Example: A visitor logbook (physical or digital, e.g., in Envoy) listing visitor names, dates, purposes (e.g., maintenance), and escort details, plus badge issuance/return records for a CDE data center.
Media Handling Procedures 9.4.1, 9.4.2, 9.4.3, 9.4.4, 9.4.5 Documented procedures for securely storing, transporting, and destroying media containing CHD (e.g., hard drives, paper records), including classification and inventory processes. Example: An SOP in Word outlining secure storage of backup tapes in a locked vault, transport via bonded courier, and cross-cut shredding of paper receipts, plus an annual media inventory log in Excel.
Media Destruction Records 9.4.6, 9.4.7 Records of secure destruction of media containing CHD (e.g., electronic media wiped, hard-copy media shredded), including methods and dates. Example: A destruction log in PDF detailing wiped hard drives (e.g., using DBAN) and shredded paper records (e.g., via a third-party service), with dates and responsible personnel signatures.
POI Device Inventory and Inspection Records 9.5.1, 9.5.2, 9.5.3 An inventory of point-of-interaction (POI) devices (e.g., POS terminals) with make, model, location, and unique identifier, plus records of regular tampering inspections. Example: An Excel spreadsheet listing POS terminals (e.g., "Verifone VX520, Serial #12345, Store #1"), plus monthly inspection checklists in PDF confirming no tampering or skimming devices.
Physical Security Monitoring Evidence 9.1.5, 9.2.3 Evidence of monitoring mechanisms (e.g., CCTV, motion sensors) in CDE areas to detect unauthorized access, with footage/logs retained per policy. Example: CCTV configuration screenshots showing coverage of a CDE server room entrance, plus 90-day retained video logs (e.g., from Milestone XProtect) documenting access events.
