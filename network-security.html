The Network Security domain ensures the CDE is protected by robust network security controls, including firewalls, routers, and wireless configurations. The evidence below is consolidated for functional stakeholders (e.g., network, IT, security teams), using practical, real-world artifacts aligned with PCI DSS v4.0.1 requirements.

Evidence Requirement PCI DSS Requirement(s) Description and Practical Examples
Network Security Control Processes 1.1.1, 1.1.2, 1.2.1 Documented processes for installing, configuring, and maintaining network security controls (e.g., firewalls, routers), including ruleset management and review procedures. Example: A firewall management SOP in Word or SharePoint detailing configuration standards, ruleset update processes, and quarterly review schedules, with sample change logs for firewall rule updates.
Network Diagrams 1.1.2 Up-to-date diagrams showing all CDE connections, including network security controls, DMZ, and systems providing security/management services (e.g., SIEM, NTP).

Example: A Visio or Lucidchart diagram illustrating CDE servers, firewalls, DMZ, and external connections (e.g., to payment processors), with labeled segmentation boundaries.
Permitted Services Justifications 1.1.3, 1.1.4 A list of allowed services, protocols, and ports for CDE connectivity, with business justifications and security features for any deemed insecure (e.g., FTP). Example: An Excel table listing allowed protocols (e.g., HTTPS on port 443), justifications (e.g., "required for payment processing"), and compensating controls for insecure protocols (e.g., encrypted FTP with restricted access).
Firewall/Router Configurations 1.1.5, 1.2.1, 1.3.1, 1.3.2 Configuration files or screenshots showing firewall/router settings that enforce CDE security, including inbound/outbound traffic restrictions and DMZ isolation. Example: Screenshots from a firewall console (e.g., Cisco ASA, Palo Alto) showing rules blocking unauthorized inbound traffic to the CDE and restricting outbound traffic to specific destinations (e.g., acquirer IPs).
Ruleset Review Records 1.1.6 Quarterly review records of network security control rulesets, including analysis, remediation actions, and management sign-off. Example: A PDF report summarizing quarterly firewall ruleset reviews, noting removed obsolete rules, added new rules, and CISO approval signatures.
Wireless Security Configurations 1.2.3, 11.2.1 If wireless is used in or connected to the CDE, configuration standards and settings for secure wireless access points (e.g., WPA3, changed default keys). Includes rogue wireless detection scan results. Example: Screenshots of wireless AP settings (e.g., Aruba, Cisco) showing WPA3 encryption and non-default keys, plus a quarterly AirMagnet scan report confirming no unauthorized access points.
Segmentation Controls 1.3.3, 1.3.4, A3.2.3 (DESV) Documentation and configurations validating segmentation effectiveness, ensuring non-CDE systems cannot access the CDE. For DESV, includes penetration test results. Example: A report detailing firewall ACLs isolating CDE from corporate networks, plus a DESV penetration test report (e.g., from a QSA) confirming no CDE access from non-CDE systems.
