The Data Security domain ensures that cardholder data (CHD) and sensitive authentication data (SAD) are protected during storage and transmission within the Cardholder Data Environment (CDE). The evidence below is consolidated for functional stakeholders (e.g., IT, security, database administration teams), using practical, real-world artifacts aligned with PCI DSS v4.0.1 requirements.

Evidence Requirement PCI DSS Requirement(s) Description and Practical Examples
Data Retention and Disposal Policy 3.1.1, 3.2.1, 3.2.2 A documented policy specifying which CHD is retained, retention periods, storage locations, and secure disposal methods when no longer needed. Example: A PDF policy document stating PAN retention for 6 months in a payment database, with procedures for secure deletion (e.g., overwriting) and shredding of hard-copy records.
Stored Data Protection Evidence 3.3.1, 3.3.2, 3.5.1, 3.5.2 Evidence that stored PAN is rendered unreadable (e.g., via encryption, truncation, masking, or tokenization) and SAD is not stored post-authorization. Example: Database query results showing masked PAN (e.g., xxxx-xxxx-xxxx-1234), a tokenization system report (e.g., from Voltage), and a log confirming no SAD storage after transaction authorization.
Cryptographic Key Management Procedures 3.6.1, 3.7.1, 3.7.2, 3.7.3, 3.7.4, 3.7.5, 3.7.6, 3.7.7, 3.7.8 Documented procedures for managing cryptographic keys used to protect stored CHD, including generation, distribution, storage, and replacement, with limited access to key custodians. Example: A key management SOP in Word detailing key generation (e.g., using HSM), secure storage in a hardware security module, and custodian roles, plus a key inventory listing algorithms (e.g., AES-256) and expiry dates.
Key Custodian Acknowledgments 3.7.7 Signed acknowledgments from key custodians confirming their responsibilities for cryptographic key management. Example: Signed PDF forms from IT staff designated as key custodians, acknowledging their duties to securely manage encryption keys for the CDE database.
Transmission Protection Configurations 4.1.1, 4.2.1, 4.2.2 Configurations showing strong cryptography (e.g., TLS 1.3) for CHD transmission over open, public networks, with an inventory of trusted keys/certificates. Example: Web server configuration screenshots (e.g., Apache with TLS 1.3 enabled) for payment pages, plus a certificate inventory in Excel listing valid SSL/TLS certificates for CDE systems.
Data Storage Inventory 3.1.1, 3.2.3 An inventory of all locations where CHD is stored, including databases, files, and backups, ensuring storage is minimized. Example: An Excel spreadsheet listing CHD storage locations (e.g., "Payment DB, Table: Transactions, Column: PAN"), with details on encryption methods and retention periods.
